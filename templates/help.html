How to use MIMIC

{% endblock %} {% block CONTENT %}

MIMIC is a web app for planning and running authorized email simulation exercises (for example security awareness training). Use it only where you have explicit scope and approval. Misuse may violate law and policy.

Overview of the workflow

Typical order of work:

  1. Senders — add at least one SMTP identity you are allowed to send from.
  2. Mail templates (optional) — reusable message bodies you can pull into a campaign.
  3. Clusters — create groups and add people (email plus personalization fields).
  4. Campaigns — create a campaign, attach clusters, template, and sender, write subject and body, then launch.

After launch, the campaign is locked: you cannot edit it. To run again with changes, use Duplicate as new draft on the campaign page.

Sign in

Open the login page, enter your username and password, and submit. If you cannot sign in, confirm the account exists (see your deployment’s bootstrap process, for example init.py or the default user in config.py). Use Logout when finished on a shared machine.

Settings

The Settings page stores global options such as default mail server parameters and send delay. Campaign sending primarily uses the sender you pick on each campaign (see Senders), but you may still use Settings for convenience or future features.

Save all fields with one Save settings action. Use the clear checkboxes next to a field if you need to wipe a stored value.

Clusters (target groups)

Clusters are named lists of people. Each person has:

  • Email — required; must be unique within that cluster.
  • Name — optional; used in mail templates as {% raw %}{{ name }}{% endraw %}.
  • Text — optional; longer text for {% raw %}{{ text }}{% endraw %} (for example department or scenario-specific line).

Open a cluster to edit its description, add or remove people, or delete the whole cluster. Deleting a cluster removes its people from that cluster and unlinks it from campaigns that referenced it.

Mail templates

Mail templates are saved messages you can reuse. Create a template with a name and content. Edit or delete from the template’s page.

When configuring a draft campaign, you can select a template and optionally use On save, if body is empty, fill from selected template to copy the template into the campaign body once.

Template content can include the same Jinja placeholders as campaigns ({% raw %}{{ name }}, {{ text }}{% endraw %}); they are filled per recipient when the campaign is launched.

Senders (SMTP)

Senders are SMTP identities: from-address, server, port, and password. Create one or more, then edit or delete as needed.

  • Port 587 is typical (STARTTLS).
  • Port 465 uses implicit TLS (SMTP over SSL) in this app.

Each campaign picks exactly one sender for that run. Keep credentials appropriate to your environment; treat them as sensitive even for short tests.

Campaigns

Create and list

On Campaigns, give the campaign a unique name and optional description, then create it. The table shows status, approximate target count, how many recipients opened (tracking pixel) and clicked the link (distinct people per event type).

Configure a draft

Open a campaign to configure:

  • Clusters — one or more; recipients are the union of people in those clusters, with duplicate emails merged once.
  • Mail template — one template (optional helper for drafting).
  • Sender — one SMTP identity (required to launch).
  • Subject and Body — see Personalization below.

Use Save draft often. Use Delete draft only if you want to remove this campaign entirely (only while it is still a draft).

Launch

Launch campaign validates your choices, locks the campaign, records a run, creates per-recipient tracking tokens, logs a sent event per person, and attempts to send email to each address. After launch you see the frozen subject and body template (placeholders unchanged in the summary), plus which clusters, template, and sender were used.

Duplicate

Duplicate as new draft copies the campaign configuration into a new draft (including associations and subject/body text). You can edit the new draft freely. Use this to rerun a similar exercise or to iterate after a launch.

Personalized email (Jinja)

Campaign subject and body are Jinja2 templates. For each recipient, the app fills:

  • {% raw %}{{ name }}{% endraw %} — from the person’s Name on the cluster.
  • {% raw %}{{ text }}{% endraw %} — from the person’s Text field.

You can write plain text or HTML in the body. If the result looks like HTML (contains angle brackets), it is sent as HTML; otherwise lines are wrapped for readability.

Example subject: Hi {% raw %}{{ name }}{% endraw %}, quick question. Example body line: Dear {% raw %}{{ name }}{% endraw %}, … {% raw %}{{ text }}{% endraw %}.

Launching and tracking

Each outgoing message includes:

  • A click link — when opened in a browser, the app records a click event and redirects to the configured redirect URL (see CLICK_REDIRECT_URL in config.py).
  • An open pixel — when the mail client loads images, the app records an open event. Some clients block images; opens may be undercounted.

Campaign statistics count distinct people with at least one open or click, aggregated across runs for that campaign.

Public URL and tracking links

Tracking URLs are built from the address you use to launch the campaign (for example http://127.0.0.1:5000/ locally). Recipients must be able to reach that host from their mail client. For real recipients on the internet, run MIMIC behind a reachable hostname or tunnel (for example ngrok) and launch using that base URL.

Engagement callbacks (open/click) are handled by this application. A separate listener or second process for raw connections or hosted simulation pages may be added later; that does not change how you configure clusters and campaigns today.

Troubleshooting

Launch says “Selected clusters have no people”
Add people under Clusters, or select clusters that contain recipients.
Launch says a template or sender is missing
Select one mail template (if you rely on it) and one sender before launch. Create assets under Senders if the list is empty.
Some emails failed to send
Check SMTP host, port, TLS mode (587 vs 465), username/password, and that the provider allows sending from that from-address. The flash message may include the first error returned.
Opens or clicks stay at zero
Confirm tracking URLs in the received mail point to a host the recipient can reach. Try loading the link manually. Remember many clients block remote images.
I need to change a launched campaign
You cannot edit a locked campaign. Use Duplicate as new draft and launch the copy.

What is planned next

Future work may include a dedicated listener process for callbacks and hosted pages (for example simulation landing flows), plus UI and design improvements. The core flow—clusters, templates, senders, campaigns, personalization, and tracking—remains the foundation.
